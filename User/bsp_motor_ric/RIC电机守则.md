# RobotIC 电机守则

> 注意编码务必为 UTF-8 ！

## ID

- 标准格式：11位，最大0x7FF
- 扩展格式：29位，最大0x1FFF FFFF
- 如未特殊特殊说明，均默认使用标准格式

- DM 使用 0x7FF 作为电机寄存器配置广播 ID，注意不要用于其他用途

## Master ID 电机发给主控的报文

- 大疆、达妙（一拖四固件）：0x201 ~ 0x208
- VESC（扩展格式）：高位(status) 低8位(0x20 ~ 0x27)
- 达妙（原厂固件）：0x10~0x17

## CAN ID 主控控制电机的报文
- 大疆、达妙（一拖四固件）：0x200、0x1FF
- VESC（扩展格式）：高位(cmd) 低8位(0x20 ~ 0x27)
- 达妙（原厂固件）：0x00~0x07

## 代码规范

- 所有代码文件编码均使用 UTF-8
- 同一份代码文件中缩进一致 tab/空格:2 或者tap/空格:4

## 文件结构

数字代表层级，数字越大层级越高

- bsp_fdcan：层级0，为最底层 FDCAN 驱动，包含过滤器初始化等
- bsp_厂商名：层级2，为各厂商电机驱动，如bsp_dji、bsp_vesc、bsp_dm_ctrl等
- bsp_厂商名_drv：层级1，为各厂商电机底层API，如bsp_dm_drv（多数情况下，调试电机不需要直接使用此层级文件）
